PPE1 - Mise en place d'un outil de ticketing GLPI

Installer la solution de gestion de parc et de suivi d’incidents GLPI. GLPI est une application Web qui nécessite le serveur Web Apache2, PHP et une base de données comme MariaDB. À la rédaction de cette fiche savoirs, la version la plus récente de GLPI est 10.0.14 et nécessite de préférence la version 8.2 de PHP, et au minimum la version 8.0. Nous présenterons ici l’installation de GLPI sur un serveur Debian sans interface graphique.

Présentation du projet Guide utilisateur

PPE2 : Conception d'un script Powershell d'arrêt de session utilisateurs actives

Les administrateurs système utilisent des scripts pour automatiser les tâches fastidieuses. La répétition d’opérations d’administration telles que la création de comptes utilisateurs, pour ne citer que la plus classique, peut vite devenir très fastidieuse. PowerShell représente la dernière génération des langages de scripts permettant un gain de temps non négligeable.

Présentation Fichiers de scripts

PPE3 : Mise en place d'un NAS avec TrueNas

TrueNas est une distribution sous FreeBSD, qui permet de créer un NAS avec plusieurs technologies dont iSCSI, il y a d’autres solutions concurrentes OpenMediaVault, OpenFiler, NAS4Free. TrueNAS repose sur le Système de Fichiers ZFS. L’espace de stockage se compose de Pools. Les Pools sont composés d’un ou plusieurs vDevs. Les vDevs quant à eux sont composés d’un ou plusieurs disques physiques qui peuvent être montés en RAID (Nous parlerons ici de zRAID). La définition de tous ces termes est disponible la page zfs-dictionary.

Présentation du projet Environnement technique

PPE4 : Mise en place Liaison LDAP GLPI & Active Directory

LDAP (Lightweight Directory Access Protocol) est un protocole qui permet l'interrogation et la modification des services d'annuaire, basé sur TCP/IP. Nous allons établir une liaison annuaire LDAP, à l'aide d'un serveur Debian sur lequel sera hébergé GLPI, vers le serveur Active Directory.
